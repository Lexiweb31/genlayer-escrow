The interactive Merit experience is served by GitHub Pages and Vercel. This backend provides the shared marketplace registry and submits explicitly labeled server-signed demo transactions.

Bradbury demo accountsNo browser wallet is connected. Live actions require different server-side client and worker accounts; if both are not safely configured, the API disables live multi-role actions.
  • Client signer: deploy, fund, evaluate, appeal, and queue settlement.
  • Worker signer: accept terms and submit public work.
  • Payouts and refunds stay pending until an outbound transfer record is finalized and inspectable.
  • Legacy settlement contracts are read-only and must not be funded.
